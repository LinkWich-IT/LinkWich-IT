Requerimientos Técnicos

Requisitos para la implementación por parte del cliente

Para asegurar rendimiento estable y monitoreo en tiempo real, recomendamos dimensionar recursos según número de nodos y retención de historial.

Requisitos recomendados
  • Sistema Operativo (Host)
    Windows 10 Pro, Windows 11 Pro o Windows Server 2019 en adelante. Recomendado Windows Server cuando se requiere mayor estabilidad, múltiples NICs o segmentación por redes.
  • CPU / Procesamiento
    Para paquetes Básico / Estándar / Empresarial: Intel i5 / i7 (o equivalente) con 4–6+ núcleos.
    Para Corporativo: Intel i9 / Xeon (o equivalente) con énfasis en multihilo (ideal: 6–12+ núcleos).
  • Memoria RAM
    8–16 GB para Básico / Estándar / Empresarial.
    24–32 GB recomendado para Corporativo (topologías grandes + más retención histórica).
  • Almacenamiento
    Instalación aproximada de la aplicación: ~1 GB (sin contar historial).
    Recomendación: SSD con 500 GB a 1 TB (dependiendo del paquete y retención de datos).
  • Conectividad de red
    Acceso estable hacia la red a monitorear (ideal: misma LAN o rutas controladas). Si hay múltiples redes/VLANs, usar NICs adicionales o rutas para alcanzar subredes remotas.
  • Salida a Internet (sin bloqueos)
    Requerida para envío de alertas en tiempo real (correo/servicios externos) y para operar integraciones. En entornos con firewall estricto, se recomienda whitelist de destinos necesarios.
  • Servidor dedicado (recomendado)
    Soporta instalación en físico o virtualizado, pero se recomienda un servidor dedicado para garantizar rendimiento continuo y evitar interferencia con otras cargas.
Nota de dimensionamiento
Los recursos reales pueden variar según: número de nodos, frecuencia de sondeo, cantidad de interfaces, logs (syslog), y el periodo de retención de históricos y reportes.

Accesos y consideraciones
  • Acceso administrativo temporal para instalación/actualizaciones y creación de servicios.
  • Sincronización de hora (NTP) recomendada para reportes y trazabilidad de eventos.
  • Si existe antivirus/EDR estricto, considerar exclusión del directorio de instalación y logs (según políticas internas).
Puertos típicos (según módulos)
  • SNMP: UDP 161 (consulta) / UDP 162 (traps, opcional)
  • Syslog: UDP/TCP 514 (según configuración)
  • Acceso remoto/automatización: TCP 22 (SSH, opcional)
  • Acceso web a plataforma: HTTPS (puerto según despliegue)
